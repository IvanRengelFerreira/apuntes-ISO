==A test is not something you “do”, it is something you “write” and run once, twice, three times, etc.==

Esto no es programar, sino mas bien diseñar.
Las pruebas dan el diseño del programa

En TDD **el test dicta el código**. Escribes la especificación (el test) y el código simplemente "rellena" esa especificación. Esto te obliga a diseñar la API y las interfaces desde la perspectiva del usuario (el que llama a la función) antes de que exista la implementación .
